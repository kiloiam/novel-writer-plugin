# 智能建议引擎

每次操作完成后，根据当前项目状态生成上下文感知的「建议操作」，而不是每次都给出相同的静态建议。

## 状态信号采集

在生成建议前，运行轻量状态检查脚本获取信号（不需要额外加载大文件）：

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/check-status.sh <项目目录>
```

脚本输出 `key=value` 格式的状态信号，包括：

| 信号 | 脚本输出键 | 含义 |
|------|-----------|------|
| 现存章节数 | `chapter_count` | 项目进度阶段 |
| 有无大纲 | `has_outline` | 是否需要规划 |
| 世界观文件数 | `worldbuilding_count` | 是否需要构建 |
| 距上次巡检的章数 | `chapters_since_inspect` | 是否该检查一致性 |
| 过期伏笔数 | `overdue_foreshadowing` | 是否有伏笔风险 |
| 角色数 | `character_count` | 是否需要关系图谱 |
| 有无关系图谱 | `has_relationships` | 角色管理状态 |
| 刚完成的操作类型 | 当前操作本身 | 自然衔接的下一步 |
| `next_chapter_note` | `PROJECT.yaml`（已加载） | 用户是否留了备注 |

## 建议规则表

**按刚完成的操作分派建议，叠加状态信号调整：**

### 写作/续写完成后

| 状态信号 | 建议 |
|---------|------|
| 默认 | `继续写` / `回顾` |
| 有 `next_chapter_note` | `继续写`（提示备注内容） |
| 现存章节数 % 5 == 0 | `建议做一次一致性巡检（已连续写了5章）` |
| 过期伏笔 > 10 章未揭 | `有N条伏笔超过10章未揭示，建议回顾或回收` |
| 现存章节数 >= 10 且无关系图谱 | `角色渐多，建议生成关系图谱` |
| 现存章节数 == total_chapters | `已写到计划末章，建议整合导出或扩展大纲` |

### 润色完成后

| 状态信号 | 建议 |
|---------|------|
| 还有未润色的连续章节 | `继续润色第X章` |
| 全部已润色 | `可以整合导出了` |
| 默认 | `继续写` / `整合` |

### 大纲/世界观完成后

| 状态信号 | 建议 |
|---------|------|
| 有大纲无世界观 | `建议先构建世界观` |
| 有世界观无大纲 | `建议写大纲` |
| 两者都有 | `开始写第一章` |

### 脑暴完成后

| 状态信号 | 建议 |
|---------|------|
| 用户选了方案 | `直接开写` |
| 用户未选 | `选一个方案开写，或换个角度再脑暴` |

### 重写/段落重写完成后

| 状态信号 | 建议 |
|---------|------|
| 后续章节可能受影响 | `建议检查后续章节衔接` |
| 默认 | `继续写` / `回顾前后文` |

### 删除/导入完成后

| 状态信号 | 建议 |
|---------|------|
| 章节变动较大（删除或导入 ≥ 3 章） | `建议做一致性巡检` |
| 默认 | `继续写` / `回顾` |

### 撤销/恢复完成后

| 状态信号 | 建议 |
|---------|------|
| 恢复了旧版本，涉及角色/伏笔变化 | `建议检查角色档案和伏笔表` |
| 默认 | `继续写` |

### 一致性巡检完成后

| 状态信号 | 建议 |
|---------|------|
| 发现矛盾 | `修复上述矛盾` |
| 无矛盾 | `一切正常，继续写` |

### 项目初始化完成后

| 状态信号 | 建议 |
|---------|------|
| 默认 | `先做世界观` / `先写大纲` / `直接开写` |

## 建议格式

建议不超过 2 条，首选最相关的。格式示例：

```
建议操作：继续写第7章 / 一致性巡检（已连续写了5章）
```

```
建议操作：继续润色第4章（第3章刚润色完）
```

```
建议操作：回收伏笔（「古玉的秘密」已埋设12章未揭）
```
